# FluentNotepads 共享代码与壳交互规则

本文档定义 EditingEngine（共享代码）与各平台壳（WinUI、UWP）之间的交互边界和规则。

## 架构原则

### 1. 职责分离

**EditingEngine（共享代码）负责：**
- 文本编辑核心逻辑（Win2D + DirectWrite + Glyph Cache）
- 编辑器 UI 组件（工具栏、状态栏、编辑页面）
- 编辑状态管理（文档内容、光标位置、选择范围）
- Reactor 组件和 Element 定义
- 跨平台的编辑功能（撤销/重做、查找/替换、语法高亮）

**平台壳（WinUI/UWP）负责：**
- 应用程序生命周期管理
- 窗口和导航框架
- 主题和外观管理（AppThemeManager）
- 文件系统访问和持久化
- 平台特定的设置存储（SettingsManager）
- 应用打包和部署配置
- 平台特定的 Reactor 引擎初始化

## 通信机制

### 2. 依赖注入方向

```
壳 → EditingEngine（单向依赖）
```

- **共享代码不依赖壳**：EditingEngine 不应引用任何壳项目
- **壳引用共享代码**：通过 `.vcxitems` 共享项目引用
- **避免循环依赖**：使用接口和回调模式解耦

### 3. 数据传递方式

#### 3.1 Props 向下传递（推荐）
```csharp
// 壳代码
EditingPage(
    initialText: fileContent,
    filePath: currentFilePath,
    onSave: SaveFile,
    onThemeChanged: ApplyTheme
)
```

#### 3.2 回调向上通知
```csharp
// EditingEngine 定义
public record EditingPageElement(
    string InitialText,
    Action<string>? OnTextChanged = null,
    Action<string>? OnSaveRequested = null,
    Func<Task>? OnFileOpenRequested = null
) : Element;
```

#### 3.3 全局状态（谨慎使用）
```csharp
// 仅用于跨组件共享的只读配置
public static class EditorConfig
{
    public static bool EnableSyntaxHighlight { get; set; } = true;
    public static string DefaultEncoding { get; set; } = "UTF-8";
}
```

## 接口契约

### 4. 编辑器与壳的接口

#### 4.1 必需的壳能力接口（定义在 EditingEngine）

```csharp
// 文件操作接口（由壳实现）
public interface IFileOperations
{
    Task<string?> OpenFileAsync();
    Task<bool> SaveFileAsync(string path, string content);
    Task<string?> ReadFileAsync(string path);
}

// 设置接口（由壳实现）
public interface IAppSettings
{
    T GetSetting<T>(string key, T defaultValue);
    void SetSetting<T>(string key, T value);
    event Action? SettingsChanged;
}

// 主题接口（由壳实现）
public interface IThemeProvider
{
    bool IsDarkMode { get; }
    event Action? ThemeChanged;
}
```

#### 4.2 编辑器状态接口（定义在 EditingEngine）

```csharp
// 编辑器状态（供壳查询）
public interface IEditorState
{
    string CurrentText { get; }
    bool IsModified { get; }
    int CursorPosition { get; }
    string? FilePath { get; }
    string Encoding { get; }
}
```

## 事件与通知

### 5. 事件流向规则

#### 5.1 向上冒泡（EditingEngine → 壳）

```csharp
// 编辑器组件定义
public record EditingPageElement(
    // UI 事件
    Action<string>? OnTextChanged = null,
    Action? OnModified = null,
    
    // 文档操作请求
    Func<Task>? OnSaveRequested = null,
    Func<Task>? OnOpenRequested = null,
    
    // 状态变化通知
    Action<int, int>? OnCursorMoved = null,
    Action<string>? OnStatusMessage = null
) : Element;
```

#### 5.2 向下传递（壳 → EditingEngine）

```csharp
// 壳通过 Props 传递命令
EditingPage(
    text: currentText,
    saveCommand: _saveCommand,          // 外部触发保存
    undoCommand: _undoCommand,          // 外部触发撤销
    findCommand: _findCommand           // 外部触发查找
)
```

### 6. 避免的反模式

❌ **不要：** 共享代码直接访问壳的静态类
```csharp
// 错误示例（EditingEngine 中）
var theme = App.Current.RequestedTheme;  // ❌ 依赖壳
```

✅ **应该：** 通过 Props 或接口获取
```csharp
// 正确示例
public record Win2DEditorElement(
    bool IsDarkMode,  // 由壳传入
    ...
) : Element;
```

## 状态管理

### 7. 状态所有权

| 状态类型 | 所有者 | 访问方式 |
|---------|--------|---------|
| 文档内容 | EditingEngine | 内部 UseState |
| 文件路径 | 壳 | Props 传入 |
| 窗口大小 | 壳 | Props 传入 |
| 光标位置 | EditingEngine | 内部状态 + 回调通知 |
| 应用设置 | 壳 | 接口注入 |
| 编辑历史（撤销栈） | EditingEngine | 内部状态 |

### 8. 持久化规则

**EditingEngine：**
- 不直接访问文件系统
- 不直接访问 LocalSettings
- 通过回调请求壳进行持久化

**壳：**
- 响应保存回调，执行实际的文件写入
- 管理最近文件列表
- 保存窗口状态和用户偏好

## 平台差异处理

### 9. 条件编译策略

**共享代码：**
```csharp
// 仅在必要时使用条件编译
#if WINDOWS_UWP
    // UWP 特定的 Win2D 初始化
#else
    // WinUI 3 路径
#endif
```

**原则：**
- 最小化 `#if` 使用
- 优先通过接口抽象平台差异
- 将平台特定代码放在壳中

### 10. Reactor 引擎差异

**WinUI 3 壳：**
```csharp
ReactorApp.Run<EditingPage>(() => new ReactorAppBuilder()
    .UseWinUI3()
    .Build());
```

**UWP 壳：**
```csharp
ReactorApp.Run<EditingPage>(() => new ReactorAppBuilder()
    .UseReactorUWP()  // 使用 ReactorUWP.core
    .Build());
```

## 迁移与扩展

### 13. 添加新平台壳

1. 创建新的壳项目
2. 引用 EditingEngine.vcxitems
3. 实现必需接口（IFileOperations、IAppSettings、IThemeProvider）
4. 初始化 Reactor 引擎
5. 在入口组件中挂载 EditingPage

### 14. 扩展编辑器能力

**如果功能是跨平台的：**
- 在 EditingEngine 中实现
- 通过 Props 接受必要参数
- 通过回调通知壳

**如果功能是平台特定的：**
- 在对应壳中实现
- 通过接口可选注入到 EditingEngine
- 或在壳层包装编辑器组件

## 最佳实践总结

### ✅ 推荐做法

1. **单向数据流**：壳 → EditingEngine（Props），EditingEngine → 壳（Callbacks）
2. **接口隔离**：定义最小必要接口，按职责分离
3. **无状态组件优先**：EditingEngine 组件尽量纯函数化
4. **显式依赖**：通过构造函数/Props 传递依赖，不使用全局变量
5. **错误边界**：壳捕获并处理 EditingEngine 抛出的异常

### ❌ 避免做法

1. **反向依赖**：EditingEngine 引用壳项目
2. **静态耦合**：共享代码访问 `App.Current` 或全局单例
3. **隐式状态**：通过静态字段在组件间传递数据
4. **平台检测**：大量使用 `#if` 而不是接口抽象
5. **副作用泄漏**：在 Render 中直接进行 I/O 操作

## 示例：完整的交互流程

### 场景：用户打开文件并编辑

```
1. [壳] 用户点击"打开"按钮
   ↓
2. [壳] 调用 FileOpenPicker，获取文件路径
   ↓
3. [壳] 读取文件内容
   ↓
4. [壳] 更新 EditingPage Props：
   EditingPage(
       initialText: fileContent,
       filePath: filePath,
       onSave: SaveFile
   )
   ↓
5. [EditingEngine] Win2DTextEditor 渲染文本
   ↓
6. [用户] 编辑文本
   ↓
7. [EditingEngine] 触发 OnTextChanged 回调
   ↓
8. [壳] 标记为未保存状态
   ↓
9. [用户] 按 Ctrl+S
   ↓
10. [壳] 捕获快捷键，调用 EditingPage 的保存逻辑
    ↓
11. [EditingEngine] 触发 OnSaveRequested 回调
    ↓
12. [壳] 执行文件写入
    ↓
13. [壳] 更新 EditingPage Props（isModified: false）
```
