# EditingEngine 共享项目说明

## 项目类型

`EditingEngine` 是一个 **C# 共享项目** (Shared Project)，可以在多个目标项目之间共享代码。

## 文件结构

```
EditingEngine/
├── EditingEngine.shproj          ← 共享项目主文件 (Visual Studio)
├── EditingEngine.projitems       ← 共享项目内容定义
├── 编辑页面.cs                   ← 主编辑页面组件
├── Win2D文本编辑器.cs            ← Win2D 渲染引擎
├── Win2D编辑器元素处理器.cs      ← Reactor ControlDescriptor
├── Win2D编辑器扩展.cs            ← Fluent 扩展方法
├── DSL工厂.cs                   ← 工厂方法
├── 编辑器工具栏.cs               ← 工具栏组件
├── 编辑器状态栏.cs               ← 状态栏组件
├── README.md                    ← 架构文档
├── MIGRATION.md                 ← 迁移报告
└── 链接外部壳规则.md             ← 规则文档
```

## 项目文件

### EditingEngine.shproj
共享项目的主文件，定义项目 GUID 和引用 `.projitems`。

### EditingEngine.projitems
定义共享的源代码文件列表：
```xml
<ItemGroup>
  <Compile Include="$(MSBuildThisFileDirectory)编辑页面.cs" />
  <Compile Include="$(MSBuildThisFileDirectory)Win2D文本编辑器.cs" />
  <Compile Include="$(MSBuildThisFileDirectory)Win2D编辑器元素处理器.cs" />
  <Compile Include="$(MSBuildThisFileDirectory)Win2D编辑器扩展.cs" />
  <Compile Include="$(MSBuildThisFileDirectory)DSL工厂.cs" />
  <Compile Include="$(MSBuildThisFileDirectory)编辑器工具栏.cs" />
  <Compile Include="$(MSBuildThisFileDirectory)编辑器状态栏.cs" />
</ItemGroup>
```

## 使用方式

### 在项目中引用

在 `.csproj` 文件中添加：

```xml
<Import Project="..\EditingEngine\EditingEngine.projitems" Label="Shared" />
```

### 当前引用此共享项目的项目

1. **FluentNotepads.UWP** - UWP 应用
2. **FluentNotepads.WINUI** - WinUI 3 应用

## 共享项目的优势

### ✅ 优点
1. **单一代码库** - 同一份代码，多处使用
2. **编译时集成** - 代码直接编译到目标项目
3. **条件编译** - 支持 `#if` 预处理器指令
4. **完全访问** - 可以访问目标项目的所有类型

### ⚠️ 注意事项
1. **无独立输出** - 不生成 DLL，不能单独编译
2. **依赖目标项目** - 必须通过引用项目构建
3. **代码重复编译** - 每个引用项目都会编译一次

## Visual Studio 操作

### 打开共享项目
1. 在解决方案资源管理器中双击 `EditingEngine.shproj`
2. 或者右键点击 → "编辑项目文件"

### 添加新文件
1. 右键点击共享项目 → "添加" → "新建项"
2. 或直接在文件夹中创建 `.cs` 文件
3. 手动添加到 `EditingEngine.projitems`：
   ```xml
   <Compile Include="$(MSBuildThisFileDirectory)新文件.cs" />
   ```

### 查看引用此共享项目的项目
在解决方案资源管理器中展开共享项目节点，查看"引用"。

## 故障排除

### 问题：Visual Studio 无法打开共享项目

**原因：** 
- 缺少 `.shproj` 文件
- 使用了错误的项目类型 (如 `.vcxitems`)

**解决：**
1. 确保存在 `EditingEngine.shproj` 和 `EditingEngine.projitems`
2. 在解决方案文件中引用 `.shproj`：
   ```xml
   <Project Path="EditingEngine/EditingEngine.shproj" Id="..." />
   ```

### 问题：找不到共享项目中的类型

**原因：** 项目未正确引用 `.projitems`

**解决：**
在 `.csproj` 中添加：
```xml
<Import Project="..\EditingEngine\EditingEngine.projitems" Label="Shared" />
```

### 问题：重复定义错误

**原因：** 同时使用了 `<Compile Include>` 链接和 `<Import projitems>`

**解决：**
移除重复的 `<Compile Include>` 项，只保留 `<Import>`。
